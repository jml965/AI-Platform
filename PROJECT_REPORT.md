# تقرير تفصيلي — منصة البناء الذكي
### AI Smart Builder Platform
**التاريخ:** 10 مارس 2026

---

## 1. نظرة عامة على المشروع

منصة البناء الذكي هي بيئة تطوير متكاملة تعمل على الويب، تجمع بين واجهة عمل ثلاثية الألواح (Workspace) ومحرك ذكاء اصطناعي خلفي يتكون من 5 وكلاء متخصصين يعملون بالتسلسل لتحويل فكرة المستخدم إلى كود جاهز للتنفيذ.

## 4. واجهة العمل (Workspace UI)

### التخطيط ثلاثي الألواح
```
┌──────────────────────────────────────────────────────────────────┐
│                         الشريط العلوي (Topbar)                   │
├────────────┬─────────────────────────────────┬───────────────────┤
│            │                                 │                   │
│  سجلات     │       المعاينة المباشرة         │   شجرة الملفات    │
│  التنفيذ   │      (Live Preview)             │   (File Tree)     │
│            │                                 │     [طي ←]        │
│  Execution │       Preview Frame             │                   │
│  Logs      │                                 │   📁 client       │
│            │                                 │   📁 server       │
│  ● خطوة 1 │                                 │   📁 shared       │
│  ● خطوة 2 │      iframe / نتيجة             │   📁 src          │
│  ○ خطوة 3 │                                 │   📄 package.json │
│            │                                 │   📄 tsconfig     │
│            │                                 │                   │
├────────────┤                                 │                   │
│ [أمر...]   │                                 │   [🔍 بحث...]     │
└────────────┴─────────────────────────────────┴───────────────────┘
```

### مميزات الواجهة
- **مقابض السحب (Resize Handles):** بين كل لوحتين لتعديل العرض بالسحب
- **الحد الأدنى/الأقصى:** اللوحة اليسرى 200-500px، اليمنى 180-450px
- **زر طي اللوحة اليمنى:** أيقونة في أعلى يمين المعاينة لإخفاء/إظهار شجرة الملفات
- **شجرة ملفات حقيقية:** تقرأ الملفات الفعلية من القرص (ليست بيانات وهمية)
- **عرض مسطح:** مجلدات أولاً ثم ملفات، مرتبة أبجدياً
- **فلترة ذكية:** تُخفي المجلدات غير الضرورية (node_modules, .git, .cache...)

## 6. محرك الذكاء الاصطناعي — الخط الإنتاجي

### تسلسل التنفيذ (Pipeline)
```
المستخدم يُدخل فكرة
        │
        ▼
┌─────────────────┐
│  1. PlannerAgent │  ← تحليل الفكرة ووضع خطة تنفيذ
│     (تخطيط)      │     المدخل: prompt (نص الفكرة)
│                   │     المخرج: plan (خطة منظمة)
└────────┬──────────┘
         ▼
┌─────────────────┐
│  2. CoderAgent   │  ← توليد ملفات الكود
│     (ترميز)      │     المدخل: prompt
│                   │     المخرج: files [{path, content}]
└────────┬──────────┘
         ▼
┌─────────────────┐
│  2.5 FileManager │  ← كتابة الملفات على القرص
│  (كتابة ملفات)   │     يُنشئ المجلدات تلقائياً
└────────┬──────────┘
         ▼
┌─────────────────┐
│  3. ReviewerAgent│  ← مراجعة الكود واكتشاف المشاكل
│     (مراجعة)     │     المدخل: files
│                   │     المخرج: issues (قائمة ملاحظات)
└────────┬──────────┘
         ▼
┌─────────────────┐
│  4. ExecutorAgent│  ← تنفيذ الكود (npx tsx src/app.ts)
│     (تنفيذ)      │     المدخل: command, cwd
│                   │     المخرج: {success, stdout, stderr, exitCode}
└────────┬──────────┘
         ▼
    ┌────┴────┐
    │ نجح؟    │
    └────┬────┘
    نعم  │  لا
    │    ▼
    │  ┌─────────────────┐
    │  │ 5. DebuggerAgent │  ← تحليل الأخطاء واقتراح حلول
    │  │    (تصحيح)       │     المدخل: stderr
    │  │                   │     المخرج: debug (قائمة اقتراحات)
    │  └────────┬──────────┘
    │           │
    ▼           ▼
┌──────────────────┐
│   النتيجة النهائية │
│   {plan, files,    │
│    issues, result,  │
│    debug}           │
└──────────────────┘
```

### تفاصيل كل وكيل

| الوكيل | الملف | الأسطر | الوظيفة |
|--------|-------|--------|---------|
| PlannerAgent | `src/agents/planner-agent.ts` | 19 | يحلل طلب المستخدم ويخرج خطة مهام منظمة |
| CoderAgent | `src/agents/coder-agent.ts` | 17 | يولّد قائمة ملفات بالمسار والمحتوى |
| ReviewerAgent | `src/agents/reviewer-agent.ts` | 26 | يفحص الكود ويكتشف مشاكل الجودة والأخطاء |

| DebuggerAgent | `src/agents/debugger-agent.ts` | 24 | يحلل رسائل الخطأ ويقترح إصلاحات |
| FileManager | `src/lib/file-manager.ts` | 24 | يكتب الملفات المولّدة على القرص بإنشاء المجلدات |

## 7. نظام البث المباشر (SSE)

```
الخادم ──(SSE)──► المتصفح

الأحداث:
  • log    → سجل تنفيذ جديد
  • status → تحديث حالة (running/paused/done)
  • file   → ملف جديد أُنشئ
  • error  → خطأ حدث
```

- **الخطاف:** `useProjectStream.ts` يستمع للأحداث عبر EventSource
- **التحكم:** إيقاف مؤقت واستئناف عبر `POST /api/project/:id/pause|resume`
