# خريطة تدفق البيانات — Data Flow Map

---

## 1. تدفق المصادقة (Authentication Flow)

```
المستخدم ──► صفحة تسجيل الدخول
                │
                ▼
         Replit Auth (OIDC + PKCE)
                │
                ▼
         API Server يتحقق من التوكن
                │
                ▼
         إنشاء/جلب سجل المستخدم من DB
                │
                ▼
         إرسال session token للمتصفح
                │
                ▼
         توجيه إلى لوحة التحكم
```

---

## 2. تدفق إنشاء مشروع جديد (Create Project Flow)

```
المستخدم ──► "مشروع جديد" ──► POST /api/projects
                                    │
                                    ▼
                            التحقق من الصلاحيات
                                    │
                                    ▼
                            التحقق من حد المشاريع (الاشتراك)
                                    │
                                    ▼
                            إنشاء سجل في DB (projects table)
                                    │
                                    ▼
                            إنشاء مجلد المشروع في التخزين
                                    │
                                    ▼
                            إرجاع معرّف المشروع
```

## 3. تدفق بناء الموقع (Build Flow) — التدفق الأهم

```
المستخدم يكتب: "أريد موقع لمطعم فيه قائمة الطعام وصفحة حجز"
        │
        ▼
POST /api/build/start { projectId, prompt }
        │
        ▼
┌──────────────────────────────────────────┐
│ API Middleware                            │
│ 1. التحقق من المصادقة                    │
│ 2. التحقق من الصلاحيات                   │
│ 3. التحقق من حد التوكن المتبقي          │
│ 4. التحقق من حد الإنفاق                 │
└──────────────────────┬───────────────────┘
                       │
                       ▼
┌──────────────────────────────────────────┐
│ Execution Engine                         │
│                                          │
│ 1. تحليل الطلب:                         │
│    "موقع لمطعم" → يحتاج:               │
│    - index.html (الصفحة الرئيسية)       │
│    - menu.html (قائمة الطعام)           │
│    - booking.html (صفحة الحجز)          │
│    - styles.css (التنسيقات)             │
│    - script.js (التفاعلية)              │
│                                          │
│ 2. إنشاء مهام البناء في DB:            │
│    task_1: generate index.html           │
│    task_2: generate menu.html            │
│    task_3: generate booking.html         │
│    task_4: generate styles.css           │
│    task_5: generate script.js            │
│    task_6: review all files              │
│                                          │
│ 3. بدء التنفيذ التدريجي                 │
└──────────────────────┬───────────────────┘
                       │
        لكل مهمة (تدريجياً):
                       │
                       ▼
┌──────────────────────────────────────────┐
│ Code Generator Agent                     │
│                                          │
│ المدخلات:                               │
│ - وصف المكون: "صفحة رئيسية لمطعم"      │
│ - نوع الملف: HTML                       │
│ - سياق المشروع: "موقع مطعم"            │
│                                          │
│ الإرسال إلى AI API:                     │
│ - prompt مخصص + الوصف                   │
│ - حد التوكن: 4000                       │
│                                          │
│ المخرجات:                               │
│ - كود HTML                              │
│ - التوكن المستخدم: 2100                  │
└──────────────────────┬───────────────────┘
                       │
                       ▼
┌──────────────────────────────────────────┐
│ تسجيل في DB                             │
│                                          │
│ token_usage:                             │
│   user_id, project_id, agent: "codegen", │
│   tokens_in: 500, tokens_out: 1600,      │
│   cost: $0.003, timestamp                │
│                                          │
│ execution_logs:                          │
│   task_id, agent: "codegen",             │
│   status: "completed",                   │
│   duration: 3200ms,                      │
│   file: "index.html"                     │
└──────────────────────┬───────────────────┘
                       │
                       ▼
┌──────────────────────────────────────────┐
│ Code Reviewer Agent                      │
│                                          │
│ يفحص: صحة الكود، الأمان، التوافق       │
│                                          │
│ النتيجة:                                │
│ ├── passed ──► حفظ الملف                │
│ └── failed ──► Fix Agent (حد 3 محاولات) │
└──────────────────────┬───────────────────┘
                       │
                       ▼
┌──────────────────────────────────────────┐
│ تحديث المعاينة                           │
│                                          │
│ - حفظ الملف في project_files             │
│ - تحديث iframe المعاينة المباشرة        │
│ - إرسال SSE event للواجهة               │
│   { type: "file_ready", file: "index" }  │
│                                          │
│ المستخدم يرى الصفحة فوراً ◄──           │
└──────────────────────────────────────────┘
```

## 4. تدفق استهلاك التوكن (Token Usage Flow)

```
كل استدعاء AI API
        │
        ▼
Middleware يسجل:
├── tokens_input (الطلب)
├── tokens_output (الاستجابة)
├── cost = tokens × سعر النموذج
├── project_id
├── user_id
└── agent_type
        │
        ▼
حساب تراكمي:
├── مجموع المهمة الحالية
├── مجموع المشروع
├── مجموع اليوم
└── مجموع الشهر
        │
        ▼
فحص الحدود:
├── هل تجاوز حد المهمة؟ ──► إيقاف الوكيل
├── هل تجاوز حد المشروع؟ ──► إيقاف المشروع
├── هل تجاوز الحد اليومي؟ ──► إيقاف لليوم
├── هل وصل 80% من أي حد؟ ──► إشعار تحذيري
└── هل نفد الرصيد؟ ──► إيقاف + طلب تعبئة
```

## 5. تدفق الدفع والاشتراك (Payment Flow)

```
المستخدم ──► "ترقية الاشتراك" ──► صفحة الخطط
        │
        ▼
اختيار الخطة ──► POST /api/billing/checkout
        │
        ▼
إنشاء Stripe Checkout Session
        │
        ▼
توجيه إلى صفحة Stripe ──► المستخدم يدفع
        │
        ▼
Stripe Webhook ──► POST /api/billing/webhook
        │
        ▼
├── تحديث subscription في DB
├── تحديث الحدود (توكن أكثر، مشاريع أكثر)
├── إنشاء سجل في invoices
└── إرسال إشعار للمستخدم
```

## 6. تدفق إدارة الفريق (Team Flow)

```
المدير ──► "إنشاء فريق" ──► POST /api/teams
        │
        ▼
إنشاء سجل team في DB
        │
        ▼
المدير ──► "دعوة عضو" ──► POST /api/teams/{id}/invite
        │
        ▼
إرسال بريد دعوة ──► العضو يقبل ──► POST /api/teams/accept
        │
        ▼
إنشاء سجل team_member بالدور المحدد
        │
        ▼
RBAC Middleware يفحص الصلاحيات عند كل طلب:
├── المدير: كل شيء
├── المطور: CRUD المشاريع
├── المراجع: قراءة المشاريع + السجلات
└── المشاهد: قراءة المشاريع فقط
```
