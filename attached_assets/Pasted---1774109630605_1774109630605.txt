========================================
4) مسار العمل الجديد (إجباري)
========================================

أي طلب من المستخدم يمر هكذا:

User Request
→ strategic
→ execution_engine
→ specialist agent if needed
→ qa_pipeline
→ final result

أي طلب من المالك في /infra يمر هكذا:

Owner Request
→ infra_sysadmin
→ infra_builder أو infra_bugfixer أو infra_deploy أو infra_security
→ verification
→ final result

========================================
5) قواعد تنفيذ إلزامية
========================================

1. strategic ممنوع يرجع JSON إذا كان الطلب تنفيذي
2. strategic لا ينفذ أي أداة مباشرة
3. execution_engine هو الوحيد الذي:
   - يكتب ملفات
   - يعدل
   - يشغل build
   - يدفع git
   - يطلب deploy
4. أي عملية خطرة:
   - db_write
   - delete_file
   - git_push
   - trigger_deploy
   - rollback
   تحتاج approval
5. translator إن بقي:
   - صلاحياته فقط i18n / text replacement
   - ممنوع db_admin / exec_command / git_push

========================================
6) تعديل البرومبتات
========================================

A) strategic prompt
استبدل أي منطق يجبره على JSON فقط.

النص المطلوب إضافته:

"أنت وكيل فهم وتخطيط فقط.
ممنوع تنفيذ أي تعديل مباشر.
إذا كان الطلب تنفيذيًا، أرسل المهمة إلى execution_engine بصيغة واضحة:
- الهدف
- الملفات المحتملة
- نوع التعديل
- مستوى الخطورة
- هل يحتاج approval
ممنوع استخدام الأدوات التنفيذية مباشرة."

----------------------------------------

B) execution_engine prompt
أضف:

"أنت المنفذ الوحيد في النظام.
أي طلب تنفيذي يجب أن يمر منك.
تستقبل الهدف من strategic أو infra_sysadmin.
تحدد الأداة أو الوكيل المناسب.
لا تكتفي بالتحليل.
يجب أن تنتهي دائمًا بـ:
- تم التنفيذ
أو
- فشل التنفيذ مع السبب."

========================================
9) اختبارات إلزامية
========================================

اختبر هذه الحالات:

1. طلب تنفيذي من المستخدم:
"غير نص في الواجهة"
النتيجة المتوقعة:
strategic يخطط فقط
execution_engine ينفذ

2. طلب فني تحليلي:
"لماذا هذا الخطأ يحدث؟"
النتيجة:
strategic يحلل
لا يوجد تنفيذ

3. طلب deploy:
execution_engine أو infra_deploy
مع approval

4. طلب ترجمة:
translator أو tool translation فقط
بدون صلاحيات خطيرة

5. تأكد أن strategic لا يستطيع:
- write_files
- db_write
- git_push

========================================
10) المخرجات المطلوبة
========================================

أريد منك:
1. قائمة الملفات التي عدلتها
2. before/after لكل ملف مهم
3. جدول الوكلاء قبل/بعد
4. إثبات أن execution_engine أصبح المنفذ الوحيد
5. إثبات أن strategic لم يعد ينفذ
6. إثبات أن translator لم يعد يملك صلاحيات خطيرة
7. نتيجة الاختبارات الخمسة

ابدأ التنفيذ الآن.
